# Carte des documents ODWS

Ce document fournit une carte faisant autorité de tous les documents
au sein de l’Open Driver Worktime Space (ODWS).

Il définit :
- quels documents existent,
- leur finalité,
- leur couche,
- et leurs relations conceptuelles.

Ce document ne définit pas de règles ni n'effectue de calculs.
Il existe pour maintenir la clarté structurelle.

## 0. Objet de ce document

L'objectif de ce document est d'éviter :
- les duplications,

Il sert de source unique de vérité

## 1. Couches de documents ODWS

La documentation ODWS est divisée en couches.

### 1.1 Couche normative

Les documents normatifs définissent :
- ce qui est considéré comme vrai au sein d'ODWS,
- comment la réalité est définie,
- ce que tous les autres modèles doivent respecter.

Les documents normatifs sont :
- écurie,
- minime,
- résistant à l'interprétation.

### 1.2 Couche descriptive

Documents descriptifs :
- expliquer la réalité,
- décrire les conditions opérationnelles,
- fournir le contexte et le raisonnement.

Ils NE DOIVENT PAS redéfinir les déclarations normatives.

## 4. Relations conceptuelles

Conceptuellement :

- `working-time-definition.md` définit ce qu'est le travail
- `workday.md` définit la manière dont le travail est modélisé
- `waiting-standby-and-other-work.md` explique les concepts clés
- des documents descriptifs expliquent pourquoi et comment cela reflète la réalité

Aucun document ne peut inverser ce sens de dépendance.

---

## 5. Règles d'ajout de nouveaux documents

Tout nouveau document DOIT :

- déclarer sa couche (`normative` ou `descriptive`),
- définir son objet,
- indiquer sa relation avec les documents existants.

Les documents normatifs ne sont ajoutés que lorsque cela est strictement nécessaire.

---

## 6. Stabilité et évolution

Les documents normatifs devraient rester stables.

Les documents descriptifs peuvent évoluer
à mesure que la compréhension, la technologie ou le contexte changent.

Cette séparation garantit une cohérence à long terme
sans stagnation.
