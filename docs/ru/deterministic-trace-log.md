# Детерминированный журнал трассировки (DTL)

Детерминированный журнал трассировки (DTL) является обязательным нормативным компонентом.
спецификации открытого рабочего пространства драйверов (ODWS).

DTL существует для того, чтобы гарантировать, что каждое вычисление, выполняемое ODWS-совместимым
система прозрачна, воспроизводима и проверяема.

Система, которая дает результаты без детерминированного журнала трассировки.
НЕ ДОЛЖНО заявлять о совместимости с ODWS.

## 1. Основной принцип

> Ничего не происходит молча.

Любая операция, преобразование, вывод или упущение
выполняемые системой, ДОЛЖНЫ быть явно записаны.

---

## 2. Определение

Журнал детерминированной трассировки представляет собой хронологическую запись, предназначенную только для добавления.
всех важных для принятия решений шагов, предпринятых во время обработки данных.

ДТЛ позволяет:

- реконструкция вычислительного процесса
- проверка детерминизма
- аудит без доступа к исходному коду
- сравнение результатов по версиям системы

---

## 3. Обязательные свойства

DTL ДОЛЖЕН быть:

- детерминированный
- только добавление
- строго предписано
- неизменяемый после написания
- на основе ссылок (без дублирования данных)
- машиночитаемый
- читаемый человеком
- независимо от языка реализации

DTL НЕ ДОЛЖЕН:

- удалить записи
- перезаписать записи
- объединить несколько шагов в один
- выполнять неявные или недокументированные решения

## 4. Трассировка записи

Каждый шаг системы создает ровно одну запись трассировки.

### Обязательные поля:

- идентификатор трассировки
Уникальный идентификатор прогона обработки

- последовательность
Монотонный порядковый номер, начиная с 1

- временная метка
Техническая временная метка создания записи (нелегальная)

- компонент
Название системного компонента, создающего запись

- операция
Тип выполняемой операции

- входная_ссылка
Ссылка на входной объект или объекты

- выходная_ref
Ссылка на выходной объект или объекты

- решение
Описание принятого решения

- причина
Явное обоснование решения

- уверенность
Числовое значение в диапазоне 0,0–1,0, выражающее уверенность.

## 6. Требование детерминизма

За исключением технической отметки времени,

## 7. Записи вычислений по требованию

DTL ДОЛЖЕН записывать вычисления
выполняется до юридического или административного оформления.

Такие записи ДОЛЖНЫ включать:

- вычисление_состояние = ИНФОРМАЦИОННЫЙ
- предполагаемые или предлагаемые входные значения
- явное указание на то, что может произойти перерасчет

Расчеты по требованию:
- не являются предварительными предположениями,
- полностью детерминированы,
- оставаться проверяемыми постоянно.

Отсутствие заключительного правового акта
НЕ ДОЛЖНО препятствовать вычислениям или регистрации.

## 9. Ограничение цели

DTL записывает, как были приняты решения.
Он не предписывает юридической, экономической или политической интерпретации.

DTL существует для того, чтобы описывать реальность, а не для того, чтобы навязывать выводы.

DTL явно включает исправления и поправки, вносимые человеком.
как первоклассные детерминированные входные данные.
