# Режимы регулирования и контекст применения

Этот документ определяет, как режимы регулирования связаны с
Модель открытого рабочего пространства драйверов (ODWS).

ODWS сначала записывает реальность.
Соответствие нормативным требованиям оценивается отдельно.

---

## 1. Существование нескольких режимов регулирования

Продолжительность рабочего времени профессионального водителя в Европе регулируется
множество нормативных рамок.

Двумя основными режимами являются:

- **AETR** (Европейское соглашение о работе экипажей транспортных средств, осуществляющих международные автомобильные перевозки)
- **Регламент (ЕС) № 561/2006** (ES561)

Эти режимы сосуществуют и различаются определенными эксплуатационными ограничениями.

---

## 2. Применимость по территории

Применимый режим регулирования определяется
**территория выполнения работ**.

Смена режима может произойти:
- в течение продолжающегося периода работы,
- без смены работодателя,
- без изменения трудового договора.

Таким образом, ODWS рассматривает режим регулирования
как **контекстная информация**, а не как структурное ограничение.

---

## 3. Основной принцип

> ODWS ДОЛЖЕН зафиксировать, что на самом деле произошло,
> вне зависимости от того, разрешено ли это конкретным регламентом.

Регистрация реальности и оценка соответствия
являются явно разделенными проблемами.

---

## 4. Ключевое отличие: сегментация рабочих периодов

Одно существенное различие между режимами касается
количество периодов работы, разрешенных в течение календарного дня.

### 4.1 ЕСТР
- разрешает **только один период работы в календарный день**
- не допускает возобновления работы после ежедневного отдыха в тот же день

### 4.2 ES561
- позволяет **несколько периодов работы в течение календарного дня**
- позволяет разделить работу на ежедневные периоды отдыха

Ядро ODWS НЕ применяет эти ограничения.

Такие пределы МОГУТ оцениваться только
на уровнях нормативной интерпретации более высокого уровня.

---

## 5. Представление в ODWS

Режим регулирования МОЖЕТ быть представлен в виде метаданных, например:

JSON
{
"regulatory_regime": "ES561"
}

или

JSON
{
"regulatory_regime": "AETR"
}

Эти метаданные:

- не влияет на вычисления ядра,
- не ограничивает записываемую реальность,
- служит только для интерпретации и проверки.

---

## 6. Связь с компенсацией

Ограничения режима регулирования существуют только для целей безопасности дорожного движения и соблюдения требований.

Они НЕ ДОЛЖНЫ:

- сократить учетное время работы,
- исключить выполненную работу из вознаграждения,
- отвергнуть физическую реальность.

Компенсация рассчитывается на основе зафиксированной реальности работы, независимо от нормативных ограничений.
